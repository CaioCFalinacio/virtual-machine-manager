# Regras direcionadas à Inteligência Artificial para a geração de código.
Esse arquivo contém regras que a IA deve considerar antes de gerar qualquer código.

## 1. Sobre a geração

O código gerado será feito na linguagem de programação C.
Todas as eventuais novas variáveis ou funções devem ser nomeadas em inglês.
Não adicione comentários no código, a não ser que seja explicitamente pedido no prompt.

## 2. Pense antes de gerar o código

Caso esteja incerto do que fazer, pergunte ao invés de tentar adivinhar. Caso necessário, consulte o arquivo REQUIREMENTS.md na raiz do projeto para compreender o contexto do problema.

## 3. Preze pela eficiência

Sempre gere o menor código e o mais simples possível para a resolução do problema. Não especule. Não adicione nada além do que foi pedido.

## 4. Preze pela unidade do código do projeto

Conserte e/ou exclua apenas código que você mesmo gerou. Não refatore o que não está quebrado. Faça o código com o estilo que já existe no projeto. 

## 5. O que não deve ser alterado ou excluído

Os arquivos da pasta include, report e data não devem ser modificados, apenas os utilize, caso precise.
